# План тестирования программы "Журнал личных учебных достижений"

## 1. Общая информация

**Название проекта:** Журнал личных учебных достижений  
**Версия:** 1.0  
**Тип приложения:** Десктопное приложение на Python  
**Цель тестирования:** Проверить функциональность, надежность и соответствие требованиям  

## 2. Цели и задачи тестирования

### Основные цели:
1. Проверить корректность работы всех функций программы
2. Убедиться в соответствии требованиям
3. Выявить критические ошибки перед релизом

### Задачи:
- Модульное тестирование функций
- Интеграционное тестирование компонентов
- Системное тестирование рабочего процесса
- Тестирование обработки ошибок

## 3. Объем тестирования

### Включается:
- Функции работы с базой данных SQLite
- Графический интерфейс на tkinter
- Операции CRUD (создание, чтение, обновление, удаление)
- Экспорт данных в Word
- Работа с JSON-конфигурацией

### Исключается:
- Тестирование на разных операционных системах
- Нагрузочное тестирование
- Тестирование безопасности
- Юзабилити-тестирование с пользователями

## 4. Подход к тестированию

### 4.1 Типы тестирования:
1. **Функциональное тестирование** - проверка соответствия требованиям
2. **Интеграционное тестирование** - проверка взаимодействия компонентов
3. **Регрессионное тестирование** - проверка после изменений

### 4.2 Методы тестирования:
- **Черный ящик** - без знания внутренней структуры
- **Белый ящик** - с учетом внутренней логики
- **Смешанный подход** - комбинация методов

## 5. Критерии начала и окончания тестирования

### Критерии начала:
- Завершена разработка всех основных функций
- Готовы тестовые данные и среда
- Утверждены тестовые сценарии

### Критерии успешного завершения:
- Пройдено ≥95% тест-кейсов
- Исправлены все критичные дефекты
- Достигнуто приемлемое качество

## 6. Тестовые среды

### Основная среда:
- ОС: Windows 10/11
- Python: 3.8+
- БД: SQLite 3
- Библиотеки: tkinter, python-docx

### Резервная среда:
- ОС: Windows Server 2019
- Python: 3.10
- Те же библиотеки

## 7. План выполнения

### Фаза 1: Подготовка (1 день)
- Настройка тестовой среды
- Подготовка тестовых данных
- Создание тестовых сценариев

### Фаза 2: Модульное тестирование (2 дня)
- Тестирование функций БД
- Тестирование работы с файлами
- Тестирование бизнес-логики

### Фаза 3: Интеграционное тестирование (1 день)
- Тестирование GUI + БД
- Тестирование экспорта
- Тестирование конфигурации

### Фаза 4: Системное тестирование (1 день)
- Полные рабочие сценарии
- Тестирование на корректных данных
- Тестирование на некорректных данных

### Фаза 5: Регрессионное тестирование (0.5 дня)
- Повторное тестирование после исправлений
- Проверка стабильности

## 8. Риски и меры снижения

| Риск | Вероятность | Влияние | Меры снижения |
|------|-------------|---------|---------------|
| Нехватка времени | Средняя | Высокое | Приоритезация тестов |
| Изменение требований | Низкая | Среднее | Гибкое планирование |
| Проблемы с окружением | Низкая | Низкое | Резервное окружение |

## 9. Отчетность

### Ежедневно:
- Отчет о пройденных тестах
- Список обнаруженных дефектов
- Статус выполнения плана

### По завершении:
- Итоговый отчет о тестировании
- Рекомендации по качеству
- Матрица покрытия тестами
