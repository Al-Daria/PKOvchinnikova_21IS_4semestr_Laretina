# Бизнес-требования к приложению "Планировщик индивидуального образовательного маршрута"

## 1. Общие сведения

### 1.1. Наименование проекта
**Планировщик индивидуального образовательного маршрута (ИОМ)**

### 1.2. Цель проекта
Создание автономного десктопного приложения для планирования и мониторинга образовательных целей студентов высших учебных заведений.

### 1.3. Заказчик
Учебные заведения (университеты, институты, колледжи) и студенты.

### 1.4. Пользователи
- Студенты бакалавриата и магистратуры
- Кураторы учебных групп
- Методисты учебных отделов
- Преподаватели

## 2. Бизнес-цели

### 2.1. Основные цели
1. **Повышение академической успеваемости** студентов через структурированное планирование
2. **Улучшение навыков самоорганизации** и тайм-менеджмента
3. **Формирование осознанного подхода** к образовательному процессу
4. **Снижение отчисляемости** за счет раннего выявления проблем
5. **Повышение удовлетворенности** образовательным процессом

### 2.2. Ключевые показатели эффективности (KPI)
1. **Увеличение процента студентов**, успешно выполняющих учебный план
2. **Снижение количества задолженностей** по дисциплинам
3. **Увеличение среднего балла успеваемости**
4. **Повышение вовлеченности** во внеучебную деятельность
5. **Улучшение показателей трудоустройства** выпускников

## 3. Функциональные требования

### 3.1. Основные модули приложения

#### 3.1.1. Управление образовательными целями
- **CRUD операции** для целей (создание, чтение, обновление, удаление)
- **Категоризация целей** по типам (курс, проект, экзамен и др.)
- **Отслеживание статуса** (планируется, в процессе, завершено, отменено)
- **Планирование сроков** с указанием плановых и фактических дат

#### 3.1.2. Развитие навыков и компетенций
- **Формирование карты навыков** студента
- **Оценка уровня компетенций** по 5-балльной шкале
- **Выявление слабых зон** для дальнейшего развития
- **Получение рекомендаций** по улучшению компетенций

#### 3.1.3. Мотивационная система
- **Система достижений** (бейджи, награды)
- **Визуализация прогресса**
- **Статистика успеваемости**
- **Сравнение с предыдущими периодами**

#### 3.1.4. Отчетность
- **Формирование отчетов** в формате Word
- **Экспорт данных** для анализа
- **Аналитика по семестрам**
- **Прогнозирование результатов**

### 3.2. Технические требования
1. **Автономность работы** - без необходимости подключения к интернету
2. **Кроссплатформенность** - работа на Windows, macOS, Linux
3. **Локальное хранение данных** - SQLite или PostgreSQL
4. **Производительность** - отклик интерфейса не более 0.5 секунд
5. **Устойчивость** - обработка ошибок без потери данных

## 4. Бизнес-процессы

### 4.1. Основной рабочий процесс
```
Начало семестра → Постановка целей → Регулярное обновление прогресса → 
Анализ результатов → Корректировка целей → Конец семестра → Формирование отчета
```

### 4.2. Пользовательские сценарии
1. **Сценарий "Планирование семестра"**
   - Выбор специальности
   - Загрузка списка компетенций
   - Постановка целей на семестр
   - Определение ключевых навыков

2. **Сценарий "Еженедельный обзор"**
   - Обновление статуса целей
   - Добавление новых задач
   - Анализ прогресса
   - Корректировка планов

3. **Сценарий "Подготовка к сессии"**
   - Анализ готовности к экзаменам
   - Выявление пробелов в знаниях
   - Планирование подготовки
   - Мониторинг выполнения

## 5. Интеграционные требования

### 5.1. Внешние системы
1. **Учебные порталы** - импорт данных об учебном плане
2. **Электронные дневники** - синхронизация оценок
3. **Системы тестирования** - интеграция результатов тестов

### 5.2. Форматы данных
- **Импорт**: JSON, CSV, Excel
- **Экспорт**: Word, PDF, CSV
- **API**: REST для интеграции с LMS

## 6. Требования к безопасности

### 6.1. Защита данных
1. **Шифрование** локальной базы данных
2. **Аутентификация** пользователей
3. **Ролевая модель** доступа
4. **Резервное копирование** данных

### 6.2. Конфиденциальность
- **GDPR compliance** для обработки персональных данных
- **Анонимизация** данных для аналитики
- **Контроль доступа** к информации

## 7. Бизнес-правила

### 7.1. Правила планирования
1. **Максимальное количество целей** на семестр: 15
2. **Минимальный срок** для долгосрочных целей: 1 месяц
3. **Обязательная категоризация** всех целей
4. **Регулярность обновления** - не реже 1 раза в 2 недели

### 7.2. Правила оценки компетенций
1. **Шкала оценки**: от 0 до 5 баллов
2. **Минимальный уровень** для освоения компетенции: 3 балла
3. **Периодичность оценки**: раз в семестр
4. **Документирование** критериев оценки
