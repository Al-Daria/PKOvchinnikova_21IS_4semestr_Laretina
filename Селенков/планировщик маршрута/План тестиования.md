# План тестирования приложения "Планировщик индивидуального образовательного маршрута"

## 1. Введение

### 1.1. Цель документа
Определить стратегию, подходы, объем и ресурсы для тестирования приложения "Планировщик индивидуального образовательного маршрута".

### 1.2. Область тестирования
Полный цикл тестирования десктопного приложения на Python с графическим интерфейсом tkinter, включая функциональное, интеграционное, пользовательское и нагрузочное тестирование.

### 1.3. Объекты тестирования
- Графический интерфейс пользователя (GUI)
- Бизнес-логика приложения
- Работа с базой данных (SQLite/PostgreSQL)
- Генерация отчетов в Word
- Импорт/экспорт данных

## 2. Подходы и стратегия тестирования

### 2.1. Типы тестирования
1. **Функциональное тестирование**
2. **Интеграционное тестирование**
3. **Пользовательское тестирование (UX)**
4. **Нагрузочное тестирование**
5. **Тестирование безопасности**
6. **Регрессионное тестирование**

### 2.2. Методологии
- **Black-box testing** - с точки зрения пользователя
- **White-box testing** - анализ кода
- **Gray-box testing** - комбинированный подход

### 2.3. Критерии начала тестирования
- Завершение разработки всех основных модулей
- Готовность тестового окружения
- Наличие тестовых данных
- Утвержденные требования и спецификации

### 2.4. Критерии завершения тестирования
- Выполнение всех тестовых сценариев
- Уровень покрытия кода ≥ 85%
- Критические и блокирующие баги исправлены
- Средний и низкий приоритет багов ≤ 5%
- Успешное прохождение приемочного тестирования

## 5. Тестовые сценарии

### 5.1. Функциональное тестирование

#### TC-001: Создание цели
**Предусловие:** Приложение запущено, вкладка "Мои цели" активна
**Шаги:**
1. Заполнить поле "Название цели"
2. Выбрать тип цели из выпадающего списка
3. Выбрать статус цели
4. Указать плановую дату
5. Добавить навыки (до 3х)
6. Добавить компетенции с уровнями
7. Заполнить описание с форматированием
8. Нажать кнопку "Сохранить цель"
**Ожидаемый результат:** Цель сохранена в БД, появилась в списке целей

#### TC-002: Редактирование цели
**Предусловие:** В системе есть минимум одна сохраненная цель
**Шаги:**
1. Выбрать цель в списке
2. Нажать "Загрузить выбранную"
3. Изменить название цели
4. Изменить статус на "Завершено"
5. Указать фактическую дату
6. Нажать "Сохранить цель"
**Ожидаемый результат:** Изменения сохранены, статус обновлен в списке

#### TC-003: Удаление цели
**Предусловие:** В системе есть минимум одна сохраненная цель
**Шаги:**
1. Выбрать цель в списке
2. Нажать "Удалить выбранную"
3. Подтвердить удаление в диалоговом окне
**Ожидаемый результат:** Цель удалена из БД и из списка

#### TC-004: Проверка системы достижений
**Предусловие:** В системе есть несколько целей разных типов
**Шаги:**
1. Создать 5 целей в статусе "В процессе"
2. Создать цели 3 разных типов
3. Завершить 4 цели с одним навыком
4. Завершить 3 цели раньше планового срока
**Ожидаемый результат:** 
- Активировано достижение "Планировщик" (5+ целей в процессе)
- Активировано достижение "Многоцелевой" (3+ типа целей)
- Активировано достижение "Навыковый рост" (4+ цели с одним навыком)
- Активировано достижение "Пунктуальный" (3+ цели в срок)

#### TC-005: Генерация отчета
**Предусловие:** В системе есть данные о целях, навыках и компетенциях
**Шаги:**
1. Перейти на вкладку "Цели на семестр"
2. Нажать "Сформировать отчёт"
3. Проверить наличие файла отчета
4. Открыть файл в Word
**Ожидаемый результат:** Создан файл отчета в формате .docx со всеми разделами

## 6. Автоматизация тестирования

### 6.1. Инструменты автоматизации
```python
# Пример тестов с использованием unittest и pytest
import unittest
import tempfile
import os
from tkinter import Tk
from research_assistant import EducationalRoutePlanner

class TestEducationalRoutePlanner(unittest.TestCase):
    def setUp(self):
        self.root = Tk()
        self.app = EducationalRoutePlanner(self.root)
        
    def test_goal_creation(self):
        # Тест создания цели
        test_data = {
            "name": "Test Goal",
            "type": "Курс",
            "status": "В процессе"
        }
        # Выполнение теста
        result = self.app.save_goal(test_data)
        self.assertTrue(result)
        
    def test_achievements_logic(self):
        # Тест логики достижений
        goals_data = [
            {"status": "В процессе"},
            {"status": "В процессе"},
            {"status": "В процессе"},
            {"status": "В процессе"},
            {"status": "В процессе"}
        ]
        # Проверка достижения "Планировщик"
        self.app.check_achievements()
        achievements = self.app.get_achievements()
        self.assertIn("планировщик", achievements)
        
    def tearDown(self):
        self.root.destroy()

if __name__ == "__main__":
    unittest.main()
```
