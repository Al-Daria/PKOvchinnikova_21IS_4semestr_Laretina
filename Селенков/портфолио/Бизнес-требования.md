# Бизнес-требования к системе "Электронное портфолио исследователя"

## 1. Общие сведения

### 1.1 Наименование проекта
**Электронное портфолио исследователя** - система управления академическими достижениями

### 1.2 Цель проекта
Создание централизованной системы для сбора, хранения, анализа и представления научно-исследовательских достижений сотрудников учебных и научных учреждений.

### 1.3 Проблематика
- Разрозненное хранение информации о научных достижениях
- Сложности при составлении отчетов и аналитики
- Отсутствие единого формата представления данных
- Трудоемкость поиска и фильтрации информации

## 2. Бизнес-цели

### 2.1 Основные цели
1. **Автоматизация** процесса учета научных достижений
2. **Стандартизация** формата представления информации
3. **Оптимизация** времени на составление отчетов на 60%
4. **Упрощение** процедуры актуализации данных
5. **Обеспечение** целостности и безопасности данных

### 2.2 Ключевые показатели успеха (KPI)
- Увеличение скорости подготовки отчетов на 70%
- Сокращение времени поиска информации на 80%
- Увеличение полноты учета достижений на 90%
- Удовлетворенность пользователей ≥ 85%
- Снижение количества ошибок в отчетах на 95%

## 3. Функциональные требования

### 3.1 Управление записями
**BR-001**: Система должна позволять создавать записи о научных достижениях с типизацией:
- Публикации (статьи, монографии)
- Участие в конференциях
- Полученные гранты
- Преподавательская деятельность
- Прочие достижения

**BR-002**: Для каждой записи должна сохраняться информация:
- Название работы/мероприятия
- Тип достижения
- Год реализации
- Файл с детальным описанием (формат Markdown)
- Дата создания и обновления

### 3.2 Управление соавторами
**BR-003**: Система должна поддерживать учет соавторов:
- Возможность добавления нескольких соавторов к одной записи
- Автоматическое определение уникальных соавторов
- Управление списком соавторов (добавление/удаление)

### 3.3 Поиск и фильтрация
**BR-004**: Пользователь должен иметь возможность:
- Просматривать все записи в едином списке
- Сортировать записи по различным критериям (дата, название, год, тип)
- Быстро находить нужные записи через структурированное представление

### 3.4 Аналитика и отчетность
**BR-005**: Система должна автоматически генерировать статистику:
- Распределение достижений по типам
- Динамика активности по годам
- Количество уникальных соавторов
- Общее количество записей

**BR-006**: Автоматическое формирование отчетов:
- Excel-отчет с ключевыми показателями и графиками
- Word-отчет для официального представления
- Графическая визуализация данных

### 3.5 История изменений
**BR-007**: Система должна вести журнал активности:
- Фиксация всех операций (создание, обновление, удаление)
- Привязка действий к конкретным записям
- Временная метка всех операций

## 4. Нефункциональные требования

### 4.1 Производительность
**BR-008**: Время отклика системы:
- Загрузка списка записей: ≤ 2 секунд
- Создание/обновление записи: ≤ 3 секунд
- Генерация отчетов: ≤ 30 секунд

### 4.2 Надежность
**BR-009**: Доступность системы ≥ 99% в рабочее время
**BR-010**: Автоматическое резервное копирование данных раз в сутки
**BR-011**: Восстановление данных при сбоях в течение 1 часа

### 4.3 Безопасность
**BR-012**: Защита данных:
- Хранение файлов в защищенной директории
- Валидация входных данных
- Защита от SQL-инъекций

### 4.4 Удобство использования
**BR-013**: Интуитивно понятный интерфейс
**BR-014**: Минимальное обучение пользователей (≤ 1 час)
**BR-015**: Подсказки и валидация при вводе данных

### 4.5 Совместимость
**BR-016**: Поддержка ОС Windows 10 и выше
**BR-017**: Совместимость с:
- PostgreSQL 12+
- Python 3.8+
- Microsoft Office 2016+ (для открытия отчетов)

## 5. Бизнес-процессы

### 5.1 Основной процесс "Добавление достижения"
1. Исследователь вносит информацию о новом достижении
2. Система проверяет корректность данных
3. Создается запись в базе данных
4. Генерируется файл с описанием
5. Запись появляется в общем списке

### 5.2 Процесс "Формирование отчета"
1. Пользователь запрашивает генерацию отчета
2. Система собирает статистику
3. Создаются графики и диаграммы
4. Формируются файлы Excel и Word
5. Отчеты сохраняются в папке "reports"

### 5.3 Процесс "Поиск информации"
1. Пользователь задает критерии поиска
2. Система фильтрует записи
3. Предоставляется структурированный результат
4. Возможность экспорта результатов

## 6. Ограничения и допущения

### 6.1 Ограничения
- Максимальное количество записей: 10,000
- Максимальный размер файла описания: 10 МБ
- Поддерживаемые форматы файлов: .md, .txt
- Максимальное количество соавторов на запись: 50

### 6.2 Допущения
- Пользователь имеет базовые навыки работы с ПК
- Установлен PostgreSQL на локальной машине
- Доступно подключение к локальной базе данных
- Пользователь работает с одной учетной записью
