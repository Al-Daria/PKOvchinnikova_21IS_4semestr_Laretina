## 7. Тестовые сценарии

### 7.1 Основные тестовые сценарии:

#### Сценарий 1: Создание новой записи
1. Запустить приложение
2. Перейти на вкладку "Создание"
3. Заполнить обязательные поля:
   - Название: "Научная статья о ИИ"
   - Тип: "Публикация"
   - Год: "2023"
4. Добавить соавторов: "Иван Иванов", "Петр Петров"
5. Ввести описание в Markdown
6. Нажать "СОЗДАТЬ ЗАПИСЬ"
7. Проверить, что запись появилась в списке

**Ожидаемый результат:** Запись успешно создана, файл сохранен, данные записаны в БД

#### Сценарий 2: Редактирование записи
1. Выбрать запись в списке
2. Изменить название на "Обновленная статья о ИИ"
3. Изменить год на "2024"
4. Сохранить изменения
5. Проверить обновление данных

**Ожидаемый результат:** Данные обновлены в БД и интерфейсе

#### Сценарий 3: Генерация отчета
1. Перейти на вкладку "Аналитика"
2. Нажать "СФОРМИРОВАТЬ ОТЧЕТ"
3. Дождаться завершения генерации
4. Проверить создание файлов:
   - reports/portfolio_report.xlsx
   - reports/portfolio_report.docx
   - reports/type_distribution.png
   - reports/year_distribution.png

**Ожидаемый результат:** Все файлы созданы, содержат корректные данные

#### Сценарий 4: Удаление записи
1. Выбрать запись в списке
2. Нажать "Удалить"
3. Подтвердить удаление
4. Проверить, что запись удалена из списка
5. Проверить, что файл удален

**Ожидаемый результат:** Запись удалена из БД, файл удален с диска

### 7.2 Тестовые данные:

```python
valid_entries = [

## 15. Процедура документирования дефектов

### 15.1 Шаблон отчета о дефекте:
```
ID: [Уникальный идентификатор]
Название: [Краткое описание]
