# Бизнес-требования для системы портфолио и компетенций

## 1. Общая информация

### 1.1. Назначение документа
Документ содержит бизнес-требования к системе автоматизации учёта и анализа учебных, научных и профессиональных достижений пользователей (студентов, преподавателей, сотрудников).

### 1.2. Цель проекта
Создание единой платформы для систематизации портфолио достижений, оценки компетенций, постановки целей и мониторинга профессионального развития.

## 2. Бизнес-цели

### 2.1. Основные цели
1. **Централизация данных** – объединение разрозненных достижений пользователя в единую базу данных
2. **Анализ компетенций** – автоматическая оценка уровня развития профессиональных навыков
3. **Визуализация прогресса** – наглядное представление исследовательской деятельности
4. **Мотивация пользователей** – система достижений и целей для стимулирования развития
5. **Формирование отчётности** – автоматическая генерация структурированных отчётов

### 2.2. Целевая аудитория
1. **Студенты** – учёт учебных и научных достижений
2. **Преподаватели** – мониторинг прогресса студентов
3. **Научные сотрудники** – систематизация публикаций и проектов
4. **HR-специалисты** – оценка компетенций соискателей

## 3. Функциональные требования

### 3.1. Управление записями портфолио
1. **Добавление записей**:
   - Типы записей: проекты, публикации, конференции, практики, гранты
   - Обязательные поля: название, тип, дата
   - Дополнительные поля: описание, соавторы
   - Привязка ключевых слов (до 5 на запись)
   - Оценка связанных компетенций с уровнем от 1 до 5

2. **Просмотр и фильтрация**:
   - Табличный просмотр всех записей
   - Сортировка по дате, типу, названию
   - Детальный просмотр полной информации по записи

3. **Управление записями**:
   - Удаление записей
   - Валидация данных при вводе

### 3.2. Анализ и визуализация
1. **Исследовательская карта**:
   - Анализ частоты использования ключевых слов
   - Статистика по соавторам
   - Графическое представление исследовательских интересов

2. **Оценка компетенций**:
   - Расчёт среднего уровня по каждой компетенции
   - Автоматическое определение слабых зон
   - Формирование персонализированных рекомендаций

3. **Статистика**:
   - Количество записей по типам
   - Динамика активности по времени
   - Сводные показатели продуктивности

### 3.3. Система достижений и целей
1. **Достижения**:
   - Автоматическая разблокировка при выполнении условий
   - Отображение описания и даты получения
   - Категории: первый шаг, командная работа, продуктивность

2. **Цели на семестр**:
   - Постановка SMART-целей
   - Отслеживание прогресса выполнения
   - Визуализация статуса (выполнено/в процессе)

### 3.4. Экспорт и отчётность
1. **Генерация отчётов**:
   - Экспорт в формат Word (.docx)
   - Стандартизированная структура отчёта
   - Автоматическое включение статистики
   - Датирование и персонализация отчёта

### 3.5. Управление справочниками
1. **Компетенции**:
   - Загрузка из JSON-файла
   - Категоризация (технические, профессиональные, коммуникативные, личные)
   - Возможность расширения списка

2. **Ключевые слова**:
   - Автоматическое пополнение базы
   - Автодополнение при вводе
   - Учёт частоты использования

## 4. Нефункциональные требования

### 4.1. Производительность
- Время отклика интерфейса: < 1 секунда
- Обработка 1000+ записей без деградации производительности
- Экспорт отчёта до 100 страниц: < 30 секунд

### 4.2. Надёжность
- Автоматическое восстановление соединения с БД
- Резервное копирование конфигураций компетенций
- Валидация данных перед сохранением

### 4.3. Удобство использования
- Интуитивно понятный интерфейс
- Контекстные подсказки
- Минимальное обучение (< 30 минут)
- Поддержка русского языка

### 4.4. Совместимость
- ОС: Windows 10+, Linux, macOS
- Python 3.8+
- PostgreSQL 12+
- Совместимость с Microsoft Word 2013+

### 4.5. Безопасность
- Локальное хранение данных
- Валидация SQL-запросов
- Защита от инъекций через параметризованные запросы

## 5. Технические ограничения

### 5.1. Архитектурные
- Десктоп-приложение с графическим интерфейсом
- Локальная база данных PostgreSQL
- Модульная архитектура для расширения функционала

### 5.2. Интеграционные
- Возможность импорта данных из JSON
- Экспорт в стандартные форматы (Word)
- API для будущей интеграции с LMS системами

## 6. Бизнес-процессы

### 6.1. Основной процесс "Учёт достижений"
1. Пользователь добавляет информацию о достижении
2. Система валидирует данные
3. Пользователь связывает достижение с компетенциями
4. Система обновляет статистику
5. Система проверяет условия для разблокировки достижений
6. Пользователь просматривает обновлённую аналитику

### 6.2. Процесс "Анализ компетенций"
1. Система агрегирует данные по компетенциям
2. Рассчитывает средние уровни
3. Определяет зоны развития
4. Формирует рекомендации
5. Представляет результаты в наглядной форме

### 6.3. Процесс "Формирование отчёта"
1. Пользователь инициирует экспорт
2. Система собирает данные за выбранный период
3. Формирует структурированный документ
4. Применяет форматирование
5. Сохраняет файл в указанную директорию

## 7. Критерии успеха

### 7.1. Количественные
- Увеличение полноты учёта достижений на 80%
- Сокращение времени на формирование отчётов на 90%
- 100% пользователей могут освоить систему за 1 час
- Обработка > 1000 записей без потери производительности

### 7.2. Качественные
- Повышение мотивации пользователей к систематизации достижений
- Улучшение понимания своих компетенций и зон развития
- Снижение нагрузки на преподавателей при оценке прогресса
- Стандартизация формата представления портфолио

## 8. Ограничения и допущения

### 8.1. Ограничения
- Система предназначена для индивидуального использования
- Нет многопользовательского режима
- Нет облачного хранения данных
- Ограниченная мобильность (десктоп-приложение)

### 8.2. Допущения
- Пользователь имеет базовые навыки работы с ПК
- Установлена СУБД PostgreSQL
- Доступно 500 МБ свободного места на диске
- Система используется в образовательных или профессиональных целях
